── SMART HOUSE ──


SmartHouse es un proyecto final en el contexto de la asignatura Programación de Hardware Reconfigurable, que pone en práctica muchos aspectos y destrezas desarrolladas a lo largo de la ingeniería de computadores.

El objetivo de este proyecto es desarrollar un sistema de domótica para el hogar que pueda medir con precisión la cantidad de luz exterior y la calidad del aire interior utilizando sensores. Estos datos serán procesados y utilizados para controlar dispositivos como persianas y ventiladores en función de las condiciones ambientales.

Este proyecto busca crear un sistema de domótica para el hogar que mejore la calidad de vida al automatizar tareas y controlar la iluminación y la calidad del aire. Con el avance de la tecnología, los hogares se están volviendo más inteligentes, y este proyecto se suma a esta tendencia, explorando y aprendiendo sobre estas nuevas tecnologías.



El proyecto se enfoca en la creación de un sistema de domótica inteligente con sensores y actuadores. Utilizando sensores de luz y calidad del aire, el sistema toma medidas basadas en los datos recopilados. Estas medidas incluyen subir o bajar persianas según las condiciones de luz, enviar alertas al teléfono móvil del usuario en caso de problemas de calidad del aire, y activar un ventilador para mejorar la calidad del aire. El proyecto se desarrolló a lo largo de varias etapas basadas en este diseño.

Para la implementación del sistema de domótica, se llevó a cabo una selección de materiales y componentes clave, incluyendo un sensor de luz para detectar el día y la noche, un sensor de calidad del aire, un adaptador USB para la programación y conexión del ESP-32, este último encargado de enviar alertas al servidor del teléfono. Además, se utilizaron dos puentes H para controlar los motores de corriente continua, un motor DC para operar las persianas, un disipador de aire para simular el ventilador, cables de conexión, resistencias para el circuito, diodos LEDs como indicadores, una placa FPGA para programar los sistemas de control y una placa Protoboard para construir los circuitos necesarios.



El sensor de luz utiliza un convertidor analógico a digital (ADC) y dos comparadores para detectar niveles bajos y altos de luz en función de umbrales predefinidos. Los valores de umbral son aproximadamente 600 Lux para subir la persiana y 400 Lux para bajarla. Se utilizan biestables D y contadores para controlar la duración de la acción del motor antes de apagarse.

Se utilizaron varias herramientas de software para desarrollar este proyecto. En primer lugar, Vivado de Xilinx se empleó para el diseño, implementación y verificación de sistemas digitales complejos basados en dispositivos FPGA, que proporciona un entorno de alto nivel para definir la funcionalidad, sintetizar el diseño y generar archivos de configuración para la programación del FPGA (bitstream).

Además, se utilizó Arduino, una plataforma de desarrollo de hardware y software de código abierto, para programar el comportamiento del Pmod ESP32.

Multisim, una herramienta de diseño de circuitos electrónicos de National Instruments, se empleó para simular y diseñar circuitos electrónicos antes de su implementación física, proporcionando una representación visual precisa del funcionamiento del proyecto.

El desarrollo del software se dividió en fases y se diseñó un diagrama de flujo para tener un seguimiento de las distintas etapas del desarrollo del proyecto.


Utilizando la librería de IP proporcionada por Vivado, se hizo uso de un convertidor analógico-digital "xadc_wizard" para convertir señales analógicas en digitales. Se implementó un comparador genérico de N bits para procesar las señales convertidas y determinar el comportamiento de los actuadores. También se creó un sistema de control que utiliza biestables tipo D y un contador para activar los actuadores durante un período de tiempo establecido.


Una vez se han realizado todas las pruebas, simulaciones, tests y demás en el programa vivado y se han realizado las pruebas en placa, se procede como último paso del desarrollo del proyecto al diseño y construcción del prototipo.

En este caso se ha decidido crear un prototipo o maqueta que recree el sistema de una persiana automática, que a su vez, será controlada mediante el circuito que se ha creado a través del motor DC de la persiana y sus respectivos diodos LEDS (subir_persiana <= “LED AMARILLO”, bajar_persiana <= “LED AZUL”). Además se han añadido dos diodos LEDS más a la maqueta: calidad_aire_buena <= “LED VERDE” y calidad_aire_mala <= “LED ROJO”.
