── MyUPMclassRoom ──

Bienvenidos a "My UPM Class Room", un proyecto realizado en el marco de la asignatura "Fundamentos de la Ingeniería del Software". A lo largo del curso, se fueron tratando una serie de conceptos esenciales relacionados con la ingeniería del software, y el objetivo de este proyecto es aplicar todos esos conocimientos de manera práctica.

"My UPM Class Room" es una aplicación diseñada para automatizar los dispositivos IoT en un entorno de aula, como la gestión de la calefacción, el aire acondicionado, las persianas y otros sistemas relevantes. Este proyecto no solo demuestra nuestra capacidad para aplicar conocimientos técnicos, sino también nuestra capacidad para abordar desafíos del mundo real y desarrollar soluciones prácticas y aplicables.


Este proyecto se ha dividido en tres hitos fundamentales, y cada uno de ellos nos ha llevado más cerca de nuestro objetivo final. El uso de herramientas como StarUML ha sido esencial para la creación y documentación de nuestros diagramas, ayudándonos a visualizar y comunicar de manera efectiva nuestra visión del software.

Con "My UPM Class Room" se pretende desarrollar un software profesional con la máxima calidad. En este proyecto, hemos aplicado una variedad de conceptos, desde la creación de requisitos de software basados en las necesidades de una empresa o cliente, hasta el diseño de diagramas de casos de uso y clases, pasando por la elaboración de diagramas de flujo detallados para casos de uso extendidos.





El Hito 2 representa una etapa crítica en nuestro proyecto "My UPM Class Room". En esta fase, nos centramos en el diseño detallado del software, creando diagramas clave que serán fundamentales para la implementación y el despliegue de la aplicación. Los tres componentes principales de este hito son el Diagrama de Diseño, el Diagrama de Componentes y el Diagrama de Despliegue, cada uno desempeña un papel crucial en la construcción de un software sólido y eficiente.





El Hito 3 marca la etapa final de nuestro proyecto, donde llevamos a cabo la implementación del software en Java utilizando el patrón de diseño Modelo Vista Controlador (MVC). Este enfoque divide la aplicación en tres componentes: Modelo, Vista y Controlador, facilitando la organización y el mantenimiento del código.

Podéis encontrar el código fuente en mi repositorio de GitHub

En resumen, este proyecto no es simplemente una tarea académica, sino una oportunidad para adquirir experiencia práctica y comprender la importancia de los conceptos de ingeniería del software en la creación de software de calidad, mantenible y duradero. Al final de este viaje, esperamos que los lectores comprendan la importancia de nuestro trabajo y cómo puede aplicarse en situaciones del mundo real para resolver problemas y mejorar la eficiencia en diversos entornos. Gracias por acompañarnos en nuestro viaje a través del proyecto "My UPM Class Room".
