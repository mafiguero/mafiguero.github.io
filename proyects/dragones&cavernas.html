─── DRAGONES & CAVERNAS ───

Este proyecto se lleva a cabo en el contexto de la asignatura Bases de Datos. El objetivo de esta práctica fue desarrollar una base de datos para un videojuego por completo. Desde su diseño y modelización acorde a las especificaciones de la empresa que nos demanda, su creación en SQL y simulación utilizando contenedores de docker, así como su implementación en java.

Este proyecto se divide en 3 grandes Hitos relacionados con lo comentado anteriormente. En el primer hito del proyecto fue diseñar una base de datos para el videojuego "Dragones & Cavernas", lo más profesional posible, haciendo uso del modelo E-R (Entidad - Relación). Para ello puse un gran empeño junto con mis compañeros de equipo para lograr un buen modelo que cubra todas las especificaciones y requisitos que la empresa de videojuegos nos pedía.




El juego consistía en un MORPG con múltiples operaciones simultáneas, donde los jugadores pueden crear personajes personalizados con roles, adquirir armas, y participar en misiones individuales y colectivas.

El sistema incluía ciudades, personajes, armas, habilidades, pociones, enemigos y misiones. Mi tarea en este hito fue establecer la estructura de la base de datos para acomodar todos estos elementos, haciendo una bases de datos mantenible, duradera y abierta a posibles expansiones. Para la modelización se usó la herramienta draw.io




En el segundo hito de este proyecto, me centré en la creación y explotación de la base de datos que respaldará el videojuego "Dragones & Cavernas." Las principales tareas abordadas en este hito incluyeron la conversión del modelo entidad-relación en tablas reales (paso a tablas), la definición de instrucciones SQL para crear las tablas con atención a la integridad referencial, la resolución de consultas SQL, la implementación de funciones, procedimientos y triggers, y la incorporación de datos de ejemplo para verificar la funcionalidad de la base de datos.

Para la creación de tablas SQL, así como la implementación de funciones, procedimientos y triggers, se hizo uso de la herramienta gratuita MySQL WorkBench. Para su posterior simulación, montando toda la base de datos, se utilizó la herramienta de contenedores de docker.

Este hito es un paso esencial para llevar a cabo la gestión y operación de la base de datos en el contexto del juego.




En el tercer y último hito del proyecto, se abordó la programación (java) que permitiría la interacción con la base de datos generada en el hito 2. Esto se realizó desde dos perspectivas diferentes: utilizando JDBC (Java Database Connectivity) y el ORM Hibernate.

JDBC:

- Se implementaron cuatro métodos que se conectaban a la base de datos a través de MySQL Connector/J 8.0.
- Estos métodos posibilitaron la inserción de un nuevo dragón, la obtención de una lista de dragones derrotados por un escuadrón específico, la recuperación de hachas disponibles en una forja específica y la identificación de la espada que porta un guerrero.
- Estas tareas se llevaron a cabo en un proyecto Maven en IntelliJ, con Docker configurado para el servidor MySQL, y se crearon las clases Java necesarias.

Hibernate:

- Se utilizó Hibernate para interactuar con la base de datos.
- Se crearon clases Java correspondientes a las entidades del modelo Entidad-Relación del hito 1, y se configuró un archivo hibernate.cfg.xml para la conexión a una nueva base de datos.
- Se implementaron las clases para crear un mago y asignarle una "Daga de Madera" como arma predeterminada.
- Estas dos perspectivas permitieron programar la interacción con la base de datos de acuerdo con las tareas definidas en el hito 3, utilizando JDBC y Hibernate según fuera necesario.
