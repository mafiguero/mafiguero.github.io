En este reto se va a intentar simular un ataque Distribuido de Denegación de Servicio (DDoS) a una infraestructura. Con este objetivo en mente, se generará un gemelo en ns3 para mostrar el ataque diseñado. La intención es poder analizar lo que ocurre por dentro a la infraestructura cuando se está realizando el ataque: ver el tráfico que pasa por los routers, la saturación, como se le impide a los usuarios legítimos acceder, etc.






En este caso hemos creado un gemelo que representa la infraestructura de la etsisi (https://www.etsisi.upm.es/). Este gemelo cuenta con un router que está conectado a un switch (representando una red grande con varios switches), y luego este switch se conecta con el router final que une con el servidor principal.

El usuario intentará acceder a la etsisi a través del router1, pero el hacker se lo impedirá utilizando una botnet de 20 dispositivos.

El primer paso para realiza la simulación de nuestro ataque es obtener la ip de la infraestructura que vamos a atacar, para lo que utilizamos el comando ping y nslookup. Además, con el comando ping podemos ver las velocidades de respuesta de los servidores de la etsisi, información que resultará importante posteriormente cuando hagamos el código de ns3.




La topología que diseñé se componía de 5 nodos:

-n0. Es el usuario que quiere conectarse a la red.
-n1. Es el router con el que se establece conexión.
-n2. Es el switch de la red.
-n3. Es el router final que se conecta al servidor.
-n4. Es el servidor.

Además, tenemos dos tipos de conexiones, pp1 (800 Mbps y un tiempo RTT) y pp2 (2 Mbps con un tiempo RTT de 20ms). Las conexiones entre el usuario y la red, así como la conexión del switch final con el servidor son del tipo pp1 y el resto de ellas son del tipo pp2. Cabe destacar que pp es el acrónimo de Peer To Peer.
