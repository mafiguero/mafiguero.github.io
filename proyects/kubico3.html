─── PROYECTO KÚBICO3 ───

Cuando aprendí a resolver el cubo de Rubik, pensé que cualquiera era capaz de resolverlo si conocía el método y se lo habían enseñado correctamente. A lo largo de mi infancia he enseñado a muchos amigos y familiares a resolverlo de una manera muy simple en 7 simples pasos, aprendiéndose únicamente 3 simples algoritmos de movimientos. Una vez dispuse de los conocimientos suficientes de programación, me propuse hacer una aplicación que pudiera enseñar a cualquiera a resolver el cubo.

La idea de este proyecto surgió a raíz de un planteamiento en la que consideraba la posibilidad de que un programa en Python fuera capaz de reconocer los colores de las caras de un cubo de Rubik, para posteriormente, obtener los pasos que se han de seguir para resolverlo. Desde el principio, la idea no fue que el programa me dijera el menor número de movimientos para resolver el rompecabezas, ya que esos algoritmos, que son capaces de resolver el cubo de Rubik en menos de 23 movimientos (Número de Dios o God's Number), no sirven para que uno pueda aprender a resolver el cubo de Rubik, ya que no siguen un método genérico y se basan en complejos algoritmos matemáticos que utilizan reglas de la teoría de grupos. Mi idea consistía en que el programa te enseñase a resolver el cubo utilizando el método que todos aprendemos cuando comenzamos: "El método de principiantes".




Este proyecto se divide en tres partes:

Una primera parte donde el usuario toma fotografías a las caras del cubo de Rubik una vez revuelto. Dichas imágenes son procesadas por un programa en Python que reconoce los colores de las piezas que forman la cara y devuelve por cada imagen, en forma de fichero, una matriz 3x3 que contiene los colores de la cara correspondiente.

En la segunda parte se llama a un programa en C que trata todos los ficheros de matrices y los carga en una matriz 3D [6][3][3] para posteriormente llamar al algoritmo de resolución, también programado en C, que devuelve, 2 ficheros, uno con todos los movimientos que se deben hacer para resolver el cubo mediante “El método de principiantes” y otro fichero para poder deshacer el cubo y poder llegar al estado inicial “scramble”.

Por último, se llama a un programa en Unity que carga el fichero scramble que le ha devuelto el programa en C y revuelve el cubo hasta alcanzar el mismo estado que el cubo al que se le han hecho las fotos. Finalmente, el usuario verá en el programa en Unity los pasos que ha de seguir para resolverlo, indicándole en todo momento la parte en la que se encuentra del “Método de Principiantes”.




─── 1. TRATAMIENTO DE IMÁGENES ───

Para el tratamiento de imágenes fue necesario programar en Python un proceso que detectase los colores de las piezas que formaban una cara y devolviese, por cada cara, un fichero con una matriz 3x3 con los colores de dicha cara.

Para ello se usaron algunas librerías para Python como "Open CV" para el tratamiento de imágenes, "CSV" para manejo de ficheros y "Numpy" para manejo de arrays y matrices.





─── 2. ALGORITMO ───

La segunda parte del proyecto corresponde con el programa encargado del “algoritmo de resolución”. Este tomará los ficheros devueltos por el programa de Python y cargará en memoria una matriz 3D [6][3][3] (6 matrices 3x3) una por cada color, correspondiente a cada una de las caras del cubo. Una vez cargada, se llamará al algoritmo que calculará los movimientos de necesarios para la resolución del rompecabezas.

El algoritmo, al igual que el “Método Principiantes” consta de 7 partes tal y como se muestra en la imagen.





El algoritmo lo desarrollé por completo en lenguaje C utilizando librerías estándar. La única librería externa empleada es "rlutil.h" para mostrar las caras del cubo 2D a medida que se van completando las diferentes etapas del algoritmo. Esto se logra utilizando la función setBackGroundColor() de dicha librería, la cual permite cambiar el color de fondo en la consola

Para llevar a cabo algunas partes de la resolución ha sido necesario recurrir algoritmos de exploración de estados que calculan los distintos caminos que se pueden tomar, basándose en un valor heurístico para decidir cuál es el mejor camino para llegar al estado deseado.

Por ejemplo, en la parte del algoritmo de orientación de la cruz blanca el heurístico sería el número de centros que estén bien colocados (objetivo heurístico == 4) y los distintos caminos serían las distintas maneras de insertar una arista blanca, es decir, una de las piezas que conforman la cruz blanca.





─── 3. VISUALIZACIÓN ───

Para finalizar, una vez se obtenemos los movimientos que se deben realizar para completar los distintos pasos del “Método de Principiantes”, se llamará a un programa que visualizará los pasos uno por uno mediante un Cubo Virtual 3D animado y que mostrará el paso correspondiente de dicho método que se está realizando en ese momento.

Para la implementación del programa de visualización del cubo utilicé el entorno de Unity y el uso de numerosos scripts, programados en lenguaje C#.
